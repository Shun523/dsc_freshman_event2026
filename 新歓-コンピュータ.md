# コンピュータの仕組みをちょっと知ろう 💻

本日はご参加ありがとうございます！


**【本日のミッション】**
1. slackとArduino IDEのインストール
2. チュートリアル回路作成
3. コンピュータの「情報のやり取り」の仕組みを知る
4. 2台のマイコンを物理的なワイヤーで繋ぐ
5. プログラムを「手動」で実行して計算させる

---

# 1. ウォーミングアップ：LEDを光らせてみよう！

いきなり複雑な回路を作る前に、まずは基本の「LED点灯」を通して、
ブレッドボードの使い方と電気の流れ（回路）を体験しましょう。

**【配線のステップ】**
1. **電気の出口（5V）：** Arduinoの `5V` ピンにワイヤーを指し、ブレッドボードの穴に挿します。
2. **安全装置（抵抗）：** そのワイヤーの隣の穴から、**抵抗（ギザギザの部品）** を跨がせるように挿します。
   👉 *※これがないと、強い電気が流れすぎてLEDが破裂します！*
3. **光る部品（LED）：** 抵抗の先に、LEDの **足が長い方（＋）** を挿します。

---

4. **電気の帰り道（GND）：** LEDの **足が短い方（−）** からワイヤーを伸ばし、Arduinoの `GND` ピンへ繋ぎます。

**🎉 USBを繋いでLEDが光れば成功です！**
電気が「5V ➡ 抵抗 ➡ LED ➡ GND」の順に流れる「一本の道（回路）」が完成しました。ブレッドボードの裏側がどう繋がっているか、なんとなく掴めましたか？

---

# 2. コンピュータを構成する「4つの部品」

私たちが使うパソコンやスマホは、主に以下の5つの要素が連携して動いています。

1. **CPU（中央演算処理装置）：** 命令を解読して計算を行う場所。
2. **メモリ（主記憶装置）：** プログラムとデータを保管している場所。
3. **クロック：** ICを動かすための電気信号を生成する場所。
4. **I/O:** Input,outの略でボタン,LEDなどを指す

---

# 3. CPUとメモリの情報伝達

今回は特に、**CPU**と**メモリ**の情報のやり取りに注目します。
CPUとメモリを繋ぐ会話の通り道を**バス**と呼びます。



* **アドレスバス（要求の道）：** CPUが「〇〇番地のデータを見せて！」とメモリに伝える道。
* **データバス（返答の道）：** メモリが「はい、その番地にはこの数字が入っていますよ」とCPUに渡す道。

これから、この情報伝達を、物理的なワイヤーを使って皆さんの手で構築していきます！

---

# 4. 今回の機材構成

今日は「Arduino」という小さなコンピュータを2台使って、このやり取りを再現します。

* **左の基板（Arduino A）： CPU役**
  * 計算をする人。今回は私たちがボタンを押してクロックを生成して作業を進ませます。
* **右の基板（Arduino B）： メモリ役**
  * 「△と◇を足し算しろ」という手順と数字が、あらかじめ保存されています。

これから、この2つの基板の間に「アドレスバス」と「データバス」の橋渡しをしていきます。

---


# 5. 配線作業 ①： バスの構築（AとBを繋ぐ）

いよいよ配線です。まずはCPUとメモリのバスを作ります。ワイヤを用意してください。

| 役割 | Arduino A (左) | Arduino B (右) | 通信の方向 |
| :--- | :--- | :--- | :--- |
| **電気の基準** | **GNDピン** | **GNDピン** | **必須（一番最初に繋ぐ）** |
| アドレスバス | 8, 9, 10, 11番 | 8, 9, 10, 11番 | A ➡ B （「〇番地をくれ！」） |
| データバス | 4, 5, 6, 7番 | 4, 5, 6, 7番 | B ➡ A （「〇という数字です」）|

👉 **同じ数字の穴同士を、真っ直ぐに繋いでください。** 

---

# 6. 配線作業 ②： ブレッドボード，手動クロック（ボタン）の設置

1. ブレッドボードのプラス列にArduino CPUの`5V`ピンを、マイナス列に`GND`ピンをつなぐ
2. つないだプラス列からArduino メモリの`5V`につなぐ。

3. ボタンをブレッドボードの真ん中を挟むようにつける

4. プラス列から、ボタンの片側の足へワイヤーを繋ぎます。

---

5.  ボタンの反対側の足からArduino CPUの2番ピンにつなぎ，それと同じ列から抵抗を挟んでマイナス列につなぎます。

**【⚠️ なぜここに抵抗がいるの？】**
この抵抗がないと、GNDに5Vを直接つなぐとショートしてしまう！

---
# 7. 結果表示用のLEDをつくろう！

計算結果を光で確認するために、4つのLEDを並べます。

1. **CPUからのワイヤー：** Arduino Aの **`A0` `A1` `A2` `A3`** ピンからワイヤーを伸ばし、ブレッドボードに1列飛ばしで挿します。
2. **LEDの配置：** ワイヤーと同じ列に、LEDの **足が長い方（＋）** を挿し、もう片方の短い足（−）を別の列へ跨がせるように挿します。
3. **GNDへの帰り道：** LEDの短い足（−）の列から、**抵抗** を跨がせてブレッドボードのマイナス列へ繋ぎます。
---

# 7. システム起動！ プログラムを「手動」で動かそう 🎉

USBケーブルを繋いで、プログラムをArduinoに流し込みます。
あなたがボタンを1回押すごとに、先ほど繋いだ「バス」を通ってデータが呼び出されます。

---
1. **ボタン押す** ➡ メモリから「次の入力値をレジスタに読み込め」という命令が届く。
2. **ボタン押す** ➡ メモリから「4」を受け取りレジスタに[4]を入力し、LEDが「4」に切り替わる！
3. **ボタン押す** ➡ メモリから「次の入力値をレジスタの値に足せ」という命令が届く。
4. **ボタン押す** ➡ メモリから「3」を受け取りレジスタの値が「7」に切り替わる。

---

# お疲れ様でした！ 

無事に「CPU」と「メモリ」を物理的に繋ぎ、計算を実行させることができました！

**【Ex】**
今回作ったのは「4ビット」のコンピュータです。つまり **0〜15** までの情報を扱えます。
* **もっと大きな計算:** 答えが15までなら、「8 + 7」や「10 + 5」などの計算も可能です。メモリの中の数字を書き換えて試してみてください。(それより大きくなると…？)
* **新しい命令の追加:** Arduinoのプログラムを少し拡張すれば、「引き算」や「論理演算」など、自分だけのオリジナル命令を追加することもできます！
